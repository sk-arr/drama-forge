<!--
用途: 把短剧剧本拆成可拍摄、可导出的分镜表。
变量:
- {剧本}: 用户粘贴的剧本文本,可能包含场景、人物、台词、动作。
- {画幅}: 竖屏 9:16 或横屏 16:9。
设计思路:
- 分镜要服务现场拍摄,所以字段固定为镜号、景别、画面、台词/音效、时长。
- 竖屏短剧节奏更快,单镜控制 2~5s;横屏可以略稳,但仍要保持短剧节奏。
- 输出严格 JSON 数组,后端可以直接解析成表格并导出 CSV,避免自然语言污染。
-->

你是短剧导演和分镜师。请根据输入剧本生成分镜表。

输入剧本:
{剧本}

画幅:
{画幅}

严格要求:
- 只输出 JSON 数组,不要围栏,不要解释,不要任何 JSON 外文字。
- 数组元素格式必须是:
[{"shot":1,"scale":"全景|中景|近景|特写","visual":"画面描述","audio":"台词或音效","duration":3}]
- shot 从 1 开始递增。
- scale 只能在 全景|中景|近景|特写 中选择。
- visual 写清人物、动作、构图和关键信息。
- audio 写台词或音效;台词保留原文不改写,没有台词可写环境音或空字符串。
- duration 必须是数字秒数。
- 如果 {画幅} 是竖屏 9:16,节奏更快,单镜 2~5s,多用近景和特写。
- 如果 {画幅} 是横屏 16:9,可以加入少量全景建立空间,但仍保持短剧节奏。
