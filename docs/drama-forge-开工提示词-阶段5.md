# drama-forge 开工提示词 · 阶段 5(提示词库 + 历史记录 + 演示模式 + 全面打磨)

前提:阶段 4 已验收通过(八页中六页已完整)。规格依据《drama-forge-设计文档.md》§4.6、§4.7、§8。本阶段结束后,功能层面即为完成态,阶段 6 只做开源整理和演示物料。

---

## 给 Codex 的指令

继续建造 drama-forge。本阶段补齐最后两页(提示词库 `#/prompts`、历史记录 `#/history`),实现演示模式全量 mock,并做一轮全面打磨走查。

### 硬约束(违反即返工)

1. 代码只写在 `D:\drama-forge`,严禁仓库外建文件;零 npm 依赖;server 只监听 127.0.0.1:3900。
2. 复用已有模块,不重写不重构已验收代码(修 bug 和本阶段明确列出的小补丁除外)。
3. mock 数据质量按「会被面试官细看」的标准写,不许用 lorem 式凑数内容。
4. 完成后文字报告,不生成截图。

### 执行循环

逐任务:实现 → 自检(`node --check` + 启动冒烟)→ commit(`feat(p5): 任务名`)→ 下一个。卡住列方案等确认,不硬试。

### 任务清单

**T1 提示词库后端。**模板文件头部的注释块(用途/变量/设计思路)约定统一格式并逐个核对补齐。API:`GET /api/prompts` 返回全部模板 [{name, 显示名, 用途, 变量列表, content, isModified}](content 取用户版 data/prompts/ 优先,isModified=存在用户版);`POST /api/prompts/save` 写 `data/prompts/{name}.md`;`POST /api/prompts/reset` 删用户版恢复默认。保存后立即生效(提示词加载器本就运行时读取,验证一下即可)。

**T2 提示词库 UI。**按设计文档 §4.6:顶部说明条(「每个功能背后的提示词模板,可直接修改,保存立即生效。」);模板卡列表(6 个:选题灵感、文案-抖音、文案-快手、文案-小程序、分镜、周报),卡面=名称+用途一句话+变量胶囊(如 `{剧名}`)+「已自定义」徽章(有用户版时);点开卡片展开编辑区:等宽字体 textarea(高度自适应,最大 60vh 内滚动)+「保存」+「恢复默认」(恢复前 confirm);保存/恢复均 toast 反馈。

**T3 历史记录后端与补丁。**`GET /api/history?type=` 列表(摘要:id/type/title/时间/条数),`GET /api/history/{id}` 详情。**小补丁**:阶段 2 的选题生成当时未写历史,现在补上(type=ideas,title 如「选题灵感 · 抖音榜」),在报告中说明改动范围。

**T4 历史记录 UI。**按设计文档 §4.7:筛选胶囊(全部/文案/分镜/周报/选题/素材整理);列表行=类型图标+标题+时间+条数摘要,倒序;点行进详情:只读渲染,尽量复用各功能页的结果渲染逻辑(文案四组卡/分镜表/周报四段/选题三卡/文件 from→to 清单),提供「复制」「导出」(复用阶段 3/4 的导出工具);素材整理详情带「撤销」按钮(已撤销的显示灰色「已撤销」标记不可再点);空状态(还没有记录时引导去各功能页)。

**T5 演示模式全量 mock。**`lib/mock/` 下备齐高质量样例数据:
- `hot-*.json`:六个默认源各一份榜单(10 条,词条要像真的,短剧相关词条占比高些;cover 留空走渐变兜底);
- `ideas.json`:3 个选题(题目+一句话逻辑+适配题材);
- `copy.json`:完整四组(10 标题/10 钩子/3 简介/1 组标签),用「保洁阿姨竟是总裁」题材,质量要能直接拿去面试演示;
- `storyboard.json`:12~14 镜完整分镜(雨夜林晚回别墅那段的展开);
- `report.json`:四段式周报成品。
开启演示模式时:`/api/hot` 直接返回对应 mock(标注来源为示例);三个流式 AI 接口返回**假流式**(把 mock 内容按行切,每 120~250ms 推一段 SSE,观感接近真实生成);分镜接口延迟 1.5s 后返回整表;历史照常写入(title 加「(演示)」前缀)。设置页「测试连接」始终走真实请求不受演示模式影响。全局琥珀横幅与侧边栏胶囊状态在阶段 1 已有,核对所有页面均可见。

**T6 全面打磨走查(逐项过,报告里逐项打勾)。**
1. 八页逐页检查:空状态、加载态、错误态三态齐全,错误文案全部说人话(不出现 fetch failed/undefined 之类)。
2. 动效一致:卡片悬停上浮+阴影增强、按钮悬停上浮/按下缩放、页面切换淡入,时长曲线统一用 token。
3. 视觉对 token:颜色、圆角、阴影不允许出现 app.css 变量之外的硬编码值(mock 兜底渐变除外);对照桌面《drama-forge-首页预览.html》的质感基准。
4. toast 统一走 ui.js,成功绿/错误红,错误停留 6s。
5. 窗口缩到 1100px 宽不破版(侧边栏固定,主区网格自适应);更窄不作要求。
6. focus 可见:所有可交互元素有 focus-visible ring(2px 靛蓝)。
7. 控制台零报错零警告(F12 过一遍八页)。
8. 细节:浏览器标签 title「短剧工坊 drama-forge」+ favicon(渐变火苗 svg);所有外链 `rel="noopener"`;长词条/长文件名溢出省略号。

**T7 冒烟与收尾。**核心手测:提示词改一个字保存→立刻生成一次验证生效→恢复默认;历史各类型点开渲染正确、再导出可用、素材撤销联动;**断网+开演示模式,从首页热点到选题到文案到分镜到周报全流程走通且观感流畅**(这是面试保底路径,必须丝滑);关演示模式配真 key 一切照旧。输出文字报告(含打磨清单逐项结果)。

### 阶段验收(用户执行)

重点验一条黄金路径:关 WiFi、开演示模式,把「首页热点 → 生成选题 → 带入文案工厂流式生成 → 分镜 → 周报 → 提示词库 → 历史记录」完整走一遍,全程无报错、无生硬感、mock 内容拿得出手。再开 WiFi 关演示模式,用真 key 抽查文案和分镜。通过后进入阶段 6(README、GitHub、演示物料,收官)。
