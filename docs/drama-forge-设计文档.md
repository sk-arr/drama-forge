## 0. 项目定位(为什么做)

- 用途:江苏梦嘉控股集团「AI 效率工具研发」岗位面试演示项目。面试时间 2026-07-08 周三 14:00,现场演示,自带笔记本。

- 一句话定位:围绕短剧生产—投放—团队管理三个环节的 AI 效率工具箱。
- 演示核心话术:「短剧行业制作成本降 90%、投流成本涨 100%,效率工具的价值就是让同样的预算测更多素材、更快迭代。」

## 2. 技术架构(硬约束)

- **零依赖 Node.js**:只用 Node 内置模块(http、fs、path、crypto 等),**不装任何 npm 包**。Node ≥ 18(要用内置 fetch)。这是卖点:面试官 clone 下来 `node server.js` 就能跑,无 install 环节。
- 后端:`server.js` 单入口 + `lib/` 模块(ai.js、sources/、files.js、history.js、config.js)。静态文件服务 + `/api/*` JSON 接口。**只绑定 127.0.0.1**,不对外网开放。
- 前端:原生 HTML/CSS/JS,单页应用,hash 路由(`#/hot`、`#/storyboard`…),无框架无构建。`public/` 目录:`index.html`、`css/app.css`、`js/`(每页一个模块 + 公共 ui.js/api.js)。
- 所有对外请求(DeepSeek、热榜接口)**一律走后端转发**:避免浏览器 CORS,且 key 永不暴露给前端页面源码。
- 数据目录:`data/`(运行时生成,git 忽略)。`data/config.json`(设置+key)、`data/cache/`(热榜缓存)、`data/history/`(生成历史)、`data/prompts/`(用户改过的提示词)。
- 启动方式:`node server.js` 或双击 `start.bat`(内容:启动 server 并自动打开浏览器)。
- Windows 注意:文件路径处理全用 `path` 模块;终端相关无(本项目不开终端)。

### 3.1 设计 token(写成 CSS 变量,集中在 app.css 顶部)

## 4. 页面规格(8 页)

### 4.1 今日热点(首页,`#/hot`)

- 页首:「今日热点」(「热点」两字渐变)+ 日期 + 「x 分钟前更新」。
- 来源胶囊排:显示设置里开启的来源(默认 6 个:抖音、快手、微博、短剧榜、小红书、B站),可横向滚动;末尾刷新按钮。
- TOP 1~3:三张封面卡(封面图 + TOP 徽章 + 播放键悬浮圈 + 标题 + 热度)。封面用接口返回图;取不到则用渐变色块兜底(靛蓝/蓝/粉三色轮换)。
- 4~10 名:列表卡,行=排名+词条+热度+外链箭头,悬停行左缩进、标题变主色。
- 点击任意词条/封面:新标签页打开该平台对该词条的**搜索页**(不做内嵌播放,理由:防爬+签名过期+合规,面试话术已备)。跳转 URL 模板见 6.3。
- 底部「AI 选题灵感」卡:渐变描边卡,自动基于当前榜单给一句趋势判断;按钮「生成 3 个选题」→ 弹出选题结果(标题+一句话逻辑+适配题材),每个选题带「去生成文案」按钮 → 跳文案工厂并自动带入剧名/题材/卖点。
- 状态:加载=骨架屏;某源失败=显示缓存+顶部小字「使用 x 小时前缓存」;全失败且无缓存=空状态卡(插画感图标+「热点源暂不可用」+重试按钮)。

### 4.2 剧本转分镜(`#/storyboard`)

- 左栏(约 1/3):剧本粘贴大文本域(placeholder 给示例格式)、画幅下拉(竖屏 9:16 / 横屏 16:9)、主按钮「生成分镜」。
- 右栏:分镜表。列:镜号、景别(全/中/近/特)、画面描述、台词/音效、时长(秒)。表头下每行斑马纹分隔;顶部:集数标题 + 「14 镜 · 约 92s」统计 + 导出按钮。
- 生成中:右栏骨架行逐条填充(AI 返回 JSON 数组,逐条渲染)。
- 导出:CSV(UTF-8 带 BOM,Excel 直接打开中文不乱码),文件名 `分镜表_剧名_日期.csv`。
- AI 返回格式:严格 JSON 数组 `[{shot,scale,visual,audio,duration}]`,解析失败自动重试一次并提示。
- 写入历史(类型 storyboard)。

### 4.3 爆款文案工厂(`#/copy`,主角页面)

- 顶部输入卡:剧名、题材下拉(女频逆袭/战神归来/萌宝寻亲/穿越重生/都市打脸/婆媳家庭/其他自定义)、剧情卖点文本域、平台胶囊(抖音/快手/微信小程序,单选)、主按钮「生成文案」。
- 结果区四组:标题 10 条、前 3 秒钩子 10 条、简介 3 条、话题标签 1 组。标题和钩子卡默认露 3 条,「展开其余 7 条」折叠。
- 每条:勾选框 + 文本 + 单条复制图标。组标题右侧显示条数。
- 底部操作条:「复制选中 n 条」(主按钮)、「导出 CSV」(次按钮)。
- **流式输出**:生成时逐字/逐条打出(SSE),先标题后钩子后简介后标签;生成中主按钮变「生成中…」禁用,可点「停止」。
- 平台差异是真逻辑:不同平台注入不同风格指令(抖音=短句悬念前置;快手=口语化强情绪;微信小程序=剧情陈述+付费钩子),提示词模板见 §7。
- 从首页选题跳入时自动填充三个输入框。
- 写入历史(类型 copy)。

### 4.4 素材批量整理(`#/files`,纯本地,无需 key)

- 顶卡:文件夹路径输入 + 「选择…」(后端列目录的简易选择器即可)+ 绿色徽章「纯本地运行」。
- 规则:分类方式胶囊(按拍摄日期 / 按类型)+ 重命名模板输入(`{日期}_{类型}_{序号}`,支持这三个变量)。
- 预览卡:「扫描到 n 个文件 → m 个目录」统计;列表行=类型图标+原文件名+箭头+新路径/新文件名。**必须先预览,预览后「执行整理」主按钮才可点**。
- 执行:移动+重命名;生成整理日志 `data/history/files-*.json`(记录每条 旧路径→新路径);完成 toast + 「撤销本次」按钮(按日志反向移回)。
- 日期来源:文件修改时间(mtime);类型按扩展名映射(视频 mp4/mov/avi…、图片 jpg/png…、音频 mp3/m4a/wav…、其他)。
- 安全:只在所选目录内移动;重名自动加序号;绝不删除文件。
- 写入历史(类型 files)。

### 4.5 AI 周报(`#/report`)

- 左栏:文本域(标题「这周干了啥,随便记」,placeholder 口水账示例)、类型下拉(周报/日报)、主按钮「生成」。
- 右栏:预览卡,四段式:本周完成 / 数据亮点 / 下周计划 / 风险与需要支持,段标题主色。顶部日期范围自动推算 + 「复制」「导出 .md」。
- 流式输出。写入历史(类型 report)。

### 4.6 提示词库(`#/prompts`,沉淀展示页)

- 说明条:「每个功能背后的提示词模板,可直接修改,保存立即生效。」
- 列表:每个模板一张卡(选题灵感 / 文案-抖音 / 文案-快手 / 文案-小程序 / 分镜 / 周报),显示名称、用途一句话、变量说明(`{剧名}` 等)。
- 点开:抽屉或展开区,等宽字体编辑框 + 「保存」「恢复默认」。
- 实现:默认模板随仓库放 `prompts/*.md`(**这是开源仓库的展示资产**);用户修改后另存 `data/prompts/`,读取时用户版优先。恢复默认=删用户版。

### 4.7 历史记录(`#/history`)

- 筛选胶囊:全部 / 文案 / 分镜 / 周报 / 选题 / 素材整理。
- 列表行:类型图标+标题(如「爆款文案 · 保洁阿姨竟是总裁 · 抖音」)+时间+条数;点开=只读详情(复用各功能的结果渲染),可再次复制/导出;素材整理详情里有「撤销」。
- 存储:`data/history/` 每条一个 JSON(id=时间戳,含 type、input、output)。列表倒序,最多保留 200 条(超出删最旧)。

### 4.8 API 设置(`#/settings`)

- 卡 1 模型连接:服务商预设下拉(**DeepSeek(默认)/ Kimi / 通义千问 / 自定义**,预设自动填接口地址+模型名)、模型名、接口地址(OpenAI 兼容)、API Key(密码框+眼睛切换)、「测试连接」(实际调一次 1 token 请求,显示 ✓ 已连接 · 延迟 ms / ✗ 具体错误说人话)、「保存」。下方灰字:「Key 仅保存在本机 data 目录,不会上传,也不会进入代码仓库。」
- 预设值:DeepSeek `https://api.deepseek.com/v1` + `deepseek-chat`;Kimi `https://api.moonshot.cn/v1` + `moonshot-v1-8k`;通义 `https://dashscope.aliyuncs.com/compatible-mode/v1` + `qwen-plus`。
- 卡 2 演示模式:开关 + 说明「断网或未配 key 时,用内置示例数据完整走通所有功能」。开启时全局顶部显示细条琥珀色横幅「演示模式」,侧边栏状态胶囊同步。
- 卡 3 今日热点来源:九个开关(抖音、快手、微博、短剧榜、小红书、B站 默认开;百度、头条、知乎 默认关)+ 自动刷新间隔下拉(15/30/60 分钟,默认 30)。

## 5. AI 层设计

- 协议:OpenAI 兼容 `POST {base}/chat/completions`,`Authorization: Bearer {key}`。
- 流式:`stream:true`,后端转发 SSE 给前端(文案、周报、选题用流式;分镜用非流式 JSON 模式,便于整表解析)。
- 超时(工作台教训直接继承):连接 30s;流式采用**空闲超时 30s**(有数据就续命),不设总闹钟;非流式 60s。错误信息说人话(key 无效/余额不足/超时/网络不通分开提示)。
- temperature:文案/选题 0.9(要发散),分镜 0.3(要稳),周报 0.4。
- 分镜要求 JSON:提示词强制「只输出 JSON 数组,不要任何其他文字」,后端做容错解析(剥 markdown 围栏、找第一个 `[` 起截取),失败自动重试 1 次。

## 6. 热点源架构

### 6.1 适配器模式

`lib/sources/` 每个来源一个文件,统一导出 `{ id, name, fetchList() }`,`fetchList` 返回统一结构:`[{ rank, title, heat, cover?, url? }]`。`lib/sources/index.js` 注册表驱动 UI(设置页开关、首页胶囊自动生成)。**加新源=加一个文件**,这是面试演示点。

### 6.2 数据获取与缓存

- 优先方案:免费聚合热榜接口(开发时实测挑选可用者,候选:vvhan API、韩小韩、uomg、或自架 DailyHotApi 的公共实例)。每个适配器可写 2 个候选接口,主备切换。
- **短剧榜**:开工时实测。若聚合源无红果榜,降级方案=接行业短剧热度榜页面数据或内置每日手动更新的样例;界面标注数据来源。不承诺红果官方数据。
- 缓存:每源结果写 `data/cache/{source}.json` 带时间戳;30 分钟内直接用缓存;接口失败回退缓存并在界面标注「x 小时前」;首页只拉当前选中源,切换时再拉。
- 所有源请求超时 8s,失败不阻塞页面。

### 6.3 跳转 URL 模板(点击看视频)

- 抖音:`https://www.douyin.com/search/{词条}`
- 快手:`https://www.kuaishou.com/search/video?searchKey={词条}`
- 微博:`https://s.weibo.com/weibo?q={词条}`
- B站:`https://search.bilibili.com/all?keyword={词条}`
- 小红书:`https://www.xiaohongshu.com/search_result?keyword={词条}`
- 百度/头条/知乎/短剧榜:接口若带原文链接用原链,否则用各自搜索页。词条一律 encodeURIComponent。

## 7. 提示词设计(仓库资产)

模板放 `prompts/`,每个 md 文件 = 头部注释(用途、变量、设计思路)+ 正文模板。变量占位 `{剧名} {题材} {卖点} {平台} {榜单}` 等,后端字符串替换。要求每个模板写「设计思路」段(为什么这么写),这是给面试官看的方法论沉淀。清单:

1. `topic-ideas.md` 选题灵感:输入=当前榜单词条+热度,输出=3 个选题(题目+一句话逻辑+适配题材),要求紧扣榜单、避免硬蹭。
2. `copy-douyin.md` / `copy-kuaishou.md` / `copy-weixin.md`:公共骨架+平台风格差异段;输出结构固定(便于流式分组渲染):先 10 标题、再 10 钩子、再 3 简介、再标签组,组间用固定分隔标记(如 `===HOOKS===`)。
3. `storyboard.md` 分镜:输入剧本+画幅,输出 JSON 数组,字段 shot/scale/visual/audio/duration;景别限定全景/中景/近景/特写;竖屏提示节奏更快、3~5s 一镜。
4. `report.md` 周报:口水账→四段式;数字必须保留;没提到的不编造。

## 8. 演示模式(mock)

- 开关在设置页;开启后:AI 调用返回内置示例(带 300ms/条的假流式,观感接近真实);热点返回内置样例榜单(封面用本地渐变占位);素材整理本来就纯本地不受影响。
- mock 数据放 `lib/mock/*.json`,内容用本文档草图里的示例(保洁总裁文案、林晚分镜、周报等),质量要高——它们可能被面试官细看。
- 全局琥珀横幅提示,防止误留;真 key 配好且网络正常时建议关闭。

## 9. 安全与 API Key(硬规矩)

- key 只存 `data/config.json`;`data/` 整目录进 `.gitignore`;仓库任何文件不得出现真 key。
- 前端永远拿不到 key:浏览器只调 `/api/*`,由后端注入 Authorization。
- server 只监听 `127.0.0.1`。
- 设置页密码框默认掩码;「测试连接」结果不回显 key。
