# drama-forge 面试演示手册(7月8日 14:00 梦嘉)

## 一、演示动线(约 5 分钟,建议全程演示模式,稳)

开场一句话:
> 「我了解到咱们的业务是短剧短视频。这个行业现在制作成本被 AI 降了 90%,但投流成本翻倍,所以我做了一个围绕『同样预算测更多素材、更快迭代』的效率工具箱,给你们演示一下。」

1. **今日热点(30秒)**:打开首页。「做投放每天第一件事是看热点,我把 9 个平台的热榜聚合到首页,点词条直接跳平台看视频。接口有 30 分钟缓存,挂了自动回退旧数据。」
2. **AI 选题(40秒)**:点「生成 3 个选题」。「AI 基于当天榜单给出选题,每个都说明和榜单趋势的连接,不硬蹭。」
3. **文案工厂(90秒,主角)**:点选题上的「去生成文案」——「参数自动带进来了」。点生成,看流式。「10 条标题、10 条钩子、3 条简介、1 组标签一次出。」**切一个平台再生成一次**:「同一个剧,抖音版短句悬念,快手版老铁情绪,小程序版带付费钩子——差异做在提示词层。」勾选几条,点导出 CSV,**双击打开给他看**:「投手拿这个表直接开工。」
4. **分镜(40秒)**:粘一段剧本,生成 13 镜表格,指一下「台词/音效」列和总时长,导出 CSV。
5. **提示词库(40秒)**:「每个功能背后的提示词都沉淀成模板,写了设计思路,可以在应用内直接改,保存立即生效。」
6. **设置页(30秒)**:「模型是 OpenAI 兼容协议,DeepSeek、Kimi、通义,换个预设就行。Key 只存本机不进仓库,服务只监听本机。」
7. **收尾(30秒)**:打开工作台:「刚才这个项目就在我自己做的项目管理工具里,这是我平时管理所有项目的方式。」再亮 GitHub 仓库页:「代码开源了,零依赖,clone 下来 node server.js 就能跑。」

## 二、必背话术

- 为什么不内嵌播放视频:「评估过抓取视频流,技术上可行,但地址签名几小时过期、维护成本高还有合规风险。权衡后选了跳官方搜索页,稳定性优先。」
- AI 代码不稳定怎么办:「我有一套流程:设计文档定规格、分阶段建造、每个任务自检加 commit、91 个自动化测试兜底。工具里的流式超时、CSV 中文乱码、路径穿越这些坑都是这么发现和修掉的。」
- 能加别的平台吗:「每个热榜源就是一个几十行的适配器文件,用 Claude 写一个新源大概十分钟。」
- 怎么开发的:「行业调研 → 设计文档 → 分阶段提示词 → AI 建造 → 我验收。方法论都写在仓库 docs/build-log.md 里。」
- 反问(加分):「我想了解下,团队现在 AI 工具的使用情况,最想先解决的一个痛点是什么?」

## 三、三层保险

1. **手机热点**:7 号晚在家用热点完整彩排一遍(真 key 路径 + 演示模式路径各一遍)。
2. **演示模式**:现场默认就开演示模式,全程不依赖网络;面试官问真实效果,再切真 key 现场生成一次。
3. **备用录屏**:提前录 2~3 分钟完整演示视频,放桌面 + 手机各一份。录屏就按上面的动线走,开演示模式录。

## 五、推 GitHub 步骤(两条路任选)

**路 A(推荐,网页 + 命令行):**
1. 浏览器登录 github.com → 右上角 + → New repository → 名字填 `drama-forge`,Public,**不要**勾选任何初始化文件 → Create。
2. 回来告诉我你的 GitHub 用户名,我来执行推送;或者你自己在 `D:\drama-forge` 打开终端跑:
   ```
   git remote add origin https://github.com/你的用户名/drama-forge.git
   git push -u origin main
   ```
   第一次推会弹 GitHub 登录窗,浏览器授权一下就好。

**路 B(装 gh 命令行):**
1. `winget install GitHub.cli` → `gh auth login` 按提示浏览器授权。
2. 告诉我一声,我用 gh 一条命令建仓 + 推送。

推完在仓库页 Settings → 加 About 描述:「短剧生产/投放/团队管理的零依赖 AI 效率工具箱 · Zero-dependency AI toolbox for short-drama teams」。
