# drama-forge 开工提示词 · 阶段 3(爆款文案工厂 + 剧本转分镜)

前提:阶段 2 已验收通过(热点首页、AI 基础模块可用)。规格依据《drama-forge-设计文档.md》§4.2、§4.3、§5、§7。

---

## 给 Codex 的指令

继续建造 drama-forge。本阶段实现两个核心功能页:爆款文案工厂(`#/copy`,本项目主角页面,质量要求最高)和剧本转分镜(`#/storyboard`),并建立生成历史的写入机制。

### 硬约束(违反即返工)

1. 代码只写在 `D:\drama-forge`,严禁仓库外建文件;零 npm 依赖;server 只监听 127.0.0.1:3900。
2. 复用阶段 2 已落地的 `lib/ai.js`(流式/超时/提示词加载),**按它现有的实际接口用,不要重写它**;确有缺口再小步扩展并在报告说明。
3. 不修改不重构已验收代码(修 bug 除外)。key 不进日志。
4. 完成后文字报告,不生成截图。

### 执行循环

逐任务:实现 → 自检(`node --check` + 启动冒烟)→ commit(`feat(p3): 任务名`)→ 下一个。卡住列方案等确认,不硬试。

### 任务清单

**T1 提示词模板(仓库核心资产,认真写)。**新增 4 个模板,每个文件头部都要有注释块(用途/变量/设计思路,设计思路要写清为什么这样组织提示词):
- `prompts/copy-douyin.md`、`prompts/copy-kuaishou.md`、`prompts/copy-weixin.md`:公共骨架(角色设定=短剧投放操盘手;输入变量 `{剧名} {题材} {卖点}`;产出要求:10 条标题、10 条前 3 秒钩子台词、3 条简介、1 组话题标签 8~12 个)+ 各自的平台风格段(抖音=短句、悬念前置、反转钩子;快手=口语化、强情绪、老铁感;微信小程序=剧情陈述完整、结尾带追剧/付费钩子)。**输出格式强约束**:组间用固定分隔行 `===TITLES===`、`===HOOKS===`、`===INTROS===`、`===TAGS===`,组内每条一行以 `- ` 开头,不许输出任何多余说明文字——前端靠这些标记做流式分组渲染。
- `prompts/storyboard.md`:输入 `{剧本} {画幅}`;输出严格 JSON 数组 `[{"shot":1,"scale":"全景|中景|近景|特写","visual":"画面描述","audio":"台词或音效","duration":秒数}]`,只输出 JSON 不要围栏不要解释;竖屏要求节奏快、单镜 2~5s;台词保留原文不改写。

**T2 历史写入模块。**`lib/history.js`:`save(type, title, input, output)` 写 `data/history/{时间戳}.json`;`list(type?)` 倒序返回摘要;总数超 200 删最旧。本阶段只用 save(历史页面阶段 5 做),文案和分镜每次成功生成后各写一条(title 如「爆款文案 · 保洁阿姨竟是总裁 · 抖音」)。

**T3 文案工厂后端。**`POST /api/ai/copy`(流式 SSE):入参 {title, genre, selling, platform};按平台取对应模板、替换变量、走 lib/ai.js 流式;temperature 0.9。流结束后由后端完成分组解析并把结构化结果一并发给前端(最后一个 SSE 事件带 final JSON),同时写历史。

**T4 文案工厂 UI。**按设计文档 §4.3 全量实现:
- 顶部输入卡:剧名、题材下拉(女频逆袭/战神归来/萌宝寻亲/穿越重生/都市打脸/婆媳家庭/自定义…选自定义出现输入框)、剧情卖点文本域、平台单选胶囊(抖音/快手/微信小程序)、主按钮「生成文案」。
- 进入页面时检查 sessionStorage 里阶段 2 选题传来的参数,有则自动填充并清除。
- 流式渲染:文本流按分隔标记切组,逐条以列表行形式实时长出来;生成中按钮变「生成中…」并提供「停止」(中断请求,已出内容保留)。
- 结果四组卡:标题 10 条、钩子 10 条(这两卡默认露 3 条+「展开其余 n 条」)、简介 3 条、标签一组胶囊。每条=勾选框+文本+单条复制图标(复制成功 toast)。
- 底部操作条:「复制选中 n 条」(实时计数,主按钮)、「导出 CSV」(次按钮)。
- 未配 key 且未开演示模式:生成按钮点击 toast 引导去设置页。

**T5 导出工具。**`public/js/export.js`(或并入 ui.js):CSV 生成统一走这里,**UTF-8 带 BOM**(Excel 打开中文不乱码),字段含逗号引号时正确转义。文案 CSV 列:组别、序号、内容、是否勾选;文件名 `文案_剧名_YYYYMMDD.csv`。剪贴板复制用 navigator.clipboard 并有降级(execCommand)。

**T6 分镜后端。**`POST /api/ai/storyboard`(非流式,60s 超时):入参 {script, ratio};temperature 0.3;返回后做容错解析(剥 markdown 围栏、从第一个 `[` 截到最后一个 `]`、JSON.parse 失败自动原样重试 1 次,再失败返回人话错误);校验每条字段齐全,duration 转数字;成功写历史。

**T7 分镜 UI。**按设计文档 §4.2:左栏剧本文本域(placeholder 给一小段示例剧本格式)+ 画幅下拉(竖屏 9:16 默认/横屏 16:9)+ 主按钮;右栏分镜表(列:镜号/景别/画面/台词·音效/时长),生成中显示骨架行,完成后整表渲染;顶部统计「n 镜 · 约 x 秒」+ 「导出 CSV」(列同表格,文件名 `分镜表_日期.csv`);空状态(未生成时右栏引导文案)。

**T8 冒烟与收尾。**手测清单:三个平台各生成一次且风格可辨、流式逐条出现、停止可用、勾选计数正确、单条/批量复制、两种 CSV 用 Excel 打开中文正常、选题→文案参数带入、分镜对一段 300 字剧本出表且时长统计正确、坏 JSON 自动重试路径(可临时改提示词诱发验证后还原)、两类历史文件落盘。输出文字报告。

### 阶段验收(用户执行)

配好 key 后:文案工厂从填入到四组结果流式长出全程顺畅,切平台重新生成能看出文风差异,导出的 CSV 在 Excel 里列整齐中文正常;分镜页粘一段剧本能出完整表格并导出;`data/history/` 里能看到生成记录文件。通过后进入阶段 4(素材整理+周报)。
