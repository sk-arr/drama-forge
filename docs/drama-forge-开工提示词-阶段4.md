## 给 Codex 的指令

继续建造 drama-forge。本阶段实现两页:素材批量整理(`#/files`,纯本地,无需 key,会移动用户真实文件,**安全第一**)和 AI 周报(`#/report`)。

### 硬约束(违反即返工)

1. 代码只写在 `D:\drama-forge`,严禁仓库外建文件(测试用的临时文件夹建在仓库内,测完删除);零 npm 依赖;server 只监听 127.0.0.1:3900。
2. 素材整理**绝不删除任何文件**,只做移动和重命名,且只在用户选定的目录内操作。
3. 复用已有 lib/ai.js、lib/history.js,不重写不重构已验收代码。
4. 完成后文字报告,不生成截图。

### 执行循环

逐任务:实现 → 自检(`node --check` + 启动冒烟)→ commit(`feat(p4): 任务名`)→ 下一个。卡住列方案等确认,不硬试。

### 任务清单

**T1 文件操作后端(核心,安全设计都在这)。**`lib/files.js` + 三个 API:
- `GET /api/files/browse?path=`:返回该路径下的子文件夹列表(供前端做简易文件夹选择器);path 为空时返回各盘符根(用 `wmic` 不可靠,直接尝试 C: 到 H: 的 fs.stat 探测即可)。只列目录,不列文件内容细节。
- `POST /api/files/scan`:入参 {dir, rule(date|type), template};扫描该目录**第一层的文件**(子目录不动、不递归,报告和界面都要写明这一点);按规则生成整理计划:rule=date 用文件 mtime 的 `MMDD` 建子文件夹,rule=type 按扩展名映射(视频 mp4/mov/avi/mkv/flv,图片 jpg/jpeg/png/gif/webp,音频 mp3/m4a/wav/flac,其他)建「视频/图片/音频/其他」子文件夹;重命名模板支持 `{日期}`(YYYYMMDD)、`{类型}`、`{序号}`(按目标文件夹内 001 起);返回 `{total, dirs, plan:[{from,to}]}` 预览,不动任何文件。
- `POST /api/files/execute`:入参 {dir, plan};执行前逐条校验:from 存在、from 与 to 都必须在 dir 内(path.resolve 后前缀校验,防路径穿越)、to 已存在则自动加 `_2`、`_3` 后缀;逐条 fs.rename(跨盘不会发生,因为同目录内);单条失败记录原因继续执行;完成写日志到 lib/history(type=files,内容含完整 from/to 清单和失败清单),返回 {moved, failed, historyId}。
- `POST /api/files/undo`:入参 {historyId};按日志反向逐条移回(同样校验+冲突加后缀),结果写回该条历史标记已撤销。

**T2 素材整理 UI。**按设计文档 §4.4:顶卡(路径输入+「选择…」弹出文件夹选择器(用 browse API 逐层点入)+绿色「纯本地运行」徽章);规则卡(分类方式胶囊+重命名模板输入,模板留空=不重命名只归类);「预览」次按钮 → 预览卡(统计行「扫描到 n 个文件 → m 个目录」+ 行列表:类型图标+原名+箭头+新相对路径,超过 50 行折叠显示前 50+「等 n 条」);**「执行整理」主按钮仅在预览成功后可点**,改动任何输入后回到不可点;执行后 toast 显示结果 + 「撤销本次」按钮(调 undo);失败条目展开可见原因。空状态、加载态齐全。

**T3 周报提示词。**`prompts/report.md`(头部注释:用途/变量/设计思路):变量 `{记录} {类型} {日期范围}`;输出固定四段,段标题固定为「本周完成 / 数据亮点 / 下周计划 / 风险与需要支持」(日报时为「今日完成/数据亮点/明日计划/风险」);要求:保留原文数字不得改动、没提到的事不编造、口语转书面但不注水、每段 2~4 条以 `- ` 开头。

**T4 周报后端。**`POST /api/ai/report`(流式):入参 {text, type(week|day)};日期范围后端按当前日期推算(周报=本周一至今天,日报=今天),注入模板;temperature 0.4;流结束写历史(type=report,title 如「周报 · 6月29日—7月3日」)。

**T5 周报 UI。**按设计文档 §4.5:左栏文本域(标题「这周干了啥,随便记」,placeholder 放口水账示例)+类型下拉+主按钮「生成」;右栏预览卡:顶部日期范围+「复制」「导出 .md」,正文流式渲染,四个段标题渲染为主色小标题,条目为正文行;未生成时右栏空状态引导。导出文件名 `周报_YYYYMMDD.md`;复制为纯 markdown 文本。未配 key 且未开演示模式时 toast 引导。

**T6 冒烟与收尾。**在 `D:\drama-forge\tmp-test\` 造 15+ 个不同扩展名和 mtime 的假文件实测:按日期归类、按类型归类、带重命名模板、重名冲突加后缀、执行后撤销完整还原、路径穿越攻击(模板里塞 `..\`)被拦截;测完删除 tmp-test 并确认不在 git 里。周报:一段口水账流式生成、数字保留验证、导出 md 打开正常、历史落盘。输出文字报告(含安全校验清单逐项结果)。

### 阶段验收(用户执行)

找一个真实的杂乱文件夹(先复制一份练手):预览列表准确、执行后文件归类整齐、撤销能完整还原;周报页粘口水账能流式出四段式、里面的数字原样保留、导出复制可用。通过后进入阶段 5(提示词库+历史页+演示模式+打磨)。
