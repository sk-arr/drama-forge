## 给 Codex 的指令

继续建造 drama-forge。本阶段实现「今日热点」首页(`#/hot`):热榜源适配器、缓存降级、首页完整 UI、AI 选题灵感。规格依据《设计文档》§4.1、§5、§6、§7。

### 硬约束(与阶段 1 相同,违反即返工)

1. 代码只写在 `D:\drama-forge`,严禁在仓库外创建任何文件。
2. 零 npm 依赖;server 只监听 127.0.0.1:3900。
3. key 只从 `data/config.json` 读取,只在后端使用,任何日志和报错不得打印 key。
4. 不修改、不重构阶段 1 已验收的代码(修 bug 除外)。
5. 完成后输出文字报告,不生成截图。

### 执行循环

逐任务:实现 → 自检(`node --check` + 启动冒烟)→ commit(`feat(p2): 任务名`)→ 下一个。卡住列方案等确认,不硬试。

### 关键提醒:热榜接口必须实测

免费聚合热榜接口的可用性没有保证,**每个来源都要在开发时真实请求验证**,拿到真数据才算通。要求:
- 候选接口自行搜寻并实测(候选方向:vvhan API、韩小韩 API、uomg、DailyHotApi 的公开实例等免费热榜聚合服务)。每个适配器允许配 1 主 1 备两个接口。
- 实测通不过的来源,不许硬编码假接口糊弄;在报告里如实列出「未找到可用源」,并让该来源在界面上显示「暂不可用」空状态。
- 短剧榜(duanju)特殊:若聚合源里没有红果或短剧相关榜单,降级为内置样例榜单(标注「示例数据」),并在报告里说明。

### 任务清单

**T1 适配器框架。**`lib/sources/index.js`:注册表,导出全部源的 `{id, name, fetchList}`;统一返回结构 `[{rank, title, heat, cover, url}]`(cover/url 可空);统一请求封装(超时 8s、UA 头、JSON 解析容错)。九个源 id 与设置页开关一一对应:douyin、kuaishou、weibo、duanju、xiaohongshu、bilibili、baidu、toutiao、zhihu。

**T2 九个适配器实测接入。**每源一个文件 `lib/sources/{id}.js`。逐个实测,报告里列出每源最终使用的接口和实测结果(条数、是否带封面)。

**T3 缓存与降级。**`lib/cache.js`:每源结果写 `data/cache/{id}.json` 带 fetchedAt;设置的刷新间隔内直接回缓存;接口失败回旧缓存并附 `stale:true` 和缓存时间;无缓存且失败返回错误结构。API 路由 `GET /api/hot?source=douyin`:只拉这一个源(懒加载),返回 `{list, fetchedAt, stale}`。

**T4 首页 UI。**按设计文档 §4.1:页首(标题渐变字+日期+「x 分钟前更新」)、来源胶囊排(只显示设置里开启的源,可横向滚动,末尾刷新按钮强制绕过缓存)、TOP1~3 封面卡(接口封面图,加载失败或无封面时用三色渐变块兜底+播放键;TOP 徽章渐变)、4~10 名列表卡(悬停行左缩进+标题变主色)、加载骨架屏、stale 提示小字、空状态卡(图标+「该来源暂不可用」+重试)。切换胶囊记住上次选中(存 localStorage)。

**T5 跳转看视频。**点击封面卡或列表行,新标签页打开对应平台搜索页,URL 模板按设计文档 §6.3,词条 encodeURIComponent;接口自带原文 url 的优先用原文 url。

**T6 AI 基础设施。**`lib/ai.js`:OpenAI 兼容 chat 封装,支持非流式与流式(SSE 转发给前端);超时策略按设计文档 §5(连接 30s、流式空闲超时 30s、非流式 60s);错误说人话(key 无效/余额不足/超时/网络不通)。提示词加载器:优先 `data/prompts/{name}.md`,否则 `prompts/{name}.md`,变量 `{xxx}` 字符串替换。此模块阶段 3 会复用,接口设计干净些。

**T7 AI 选题灵感。**`prompts/topic-ideas.md` 按设计文档 §7 撰写(含头部「用途/变量/设计思路」注释)。首页底部渐变描边卡:自动显示一句基于当前榜单的趋势判断可以延后到点击时一并生成;主按钮「生成 3 个选题」→ `POST /api/ai/ideas`(流式),输入=当前源的榜单词条+热度;结果就地展开三张小卡(题目+一句话逻辑+适配题材),各带「去生成文案」按钮 → 跳 `#/copy` 并通过 sessionStorage 传递 {剧名建议、题材、卖点},文案工厂页阶段 3 接收(本阶段先把参数存好,占位页显示收到的参数即可验证)。未配 key 且未开演示模式时,点击按钮 toast 提示「先到设置页配置 API Key」。

**T8 冒烟与收尾。**手测清单:六个默认源逐个切换(可用的出数据、不可用的空状态)、断网时回缓存并显示 stale、封面缺失兜底色块、点击跳转正确、选题流式生成成功、sessionStorage 参数传递验证。输出报告:每源接口实测结果表、做了什么、遗留问题。

### 阶段验收(用户执行)

启动后:首页有真实热榜数据(至少抖音、微博、B站三源可用),TOP3 有封面或体面的兜底,点词条能跳到平台看到相关视频,配好 key 后能流式生成 3 个选题,断网刷新显示缓存。通过后进入阶段 3(文案工厂+分镜)。
